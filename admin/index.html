Deployment Guide

GitHub Pages (Recommended — Free)
1
Create a new GitHub repository. Name it yourusername.github.io for a root domain or any name for a subdomain.
2
Upload all files from the spechub2/ folder to the repository root. Include index.html, css/, js/, data/, images/, and all page folders.
3
Go to Repository Settings → Pages → Source. Set branch to main, folder to / (root). Save.
4
Your site will be live at https://yourusername.github.io within 2 minutes.
5
To use a custom domain, add a CNAME file with your domain name (e.g. spechubpro.com) and configure your DNS A records to point to GitHub Pages IPs.
Adding a New Phone (Static Workflow)
1
Use the Add Phone tab to generate the JSON entry.
2
Open data/phones.js and paste the generated entry into the phones array.
3
Create the folder phones/your-phone-id/ and add an index.html inside it. Copy an existing phone page and update all content.
4
Update sitemap.xml to include the new phone URL.
5
Commit and push all changed files to GitHub. The site updates automatically.
Other Hosting Options
Netlify / Vercel: Drag and drop the project folder onto netlify.com/drop or vercel.com. Both support custom domains for free and deploy in under 30 seconds.

Cloudflare Pages: Connect your GitHub repository to Cloudflare Pages for automatic deployments on every push.

Traditional Hosting: Upload via FTP to any web host. Configure the server to serve index.html from directories (most hosts do this by default).
